## Why insightd?

You run a homelab. You have containers everywhere. Something goes down at 2am and you don't know until morning. Grafana is overkill. Uptime Kuma only does HTTP checks. You want **one tool** that monitors everything and tells you what matters.

## Quick Start

```bash
# Single server (standalone mode)
docker run -d \
  --name insightd \
  --restart unless-stopped \
  -v /var/run/docker.sock:/var/run/docker.sock:ro \
  -v /:/host:ro \
  -v insightd-data:/data \
  -p 3000:3000 \
  andreas404/insightd-hub:0.1.0
```

Open `http://your-server:3000` and you're monitoring.

## Multi-Server Setup

```
[Agent on each host] → MQTT (Mosquitto) → [Hub] → SQLite → Web UI + Email + Webhooks
```

Deploy a Mosquitto broker + hub on one server, then add lightweight agents on each host. See the [Docker Compose guide](/insightd.dev/guides/docker-compose/) for the full setup.

## Free & Open Source

insightd is MIT licensed and free for any number of servers. No cloud dependency, no telemetry, no signup. Your data stays on your hardware.
